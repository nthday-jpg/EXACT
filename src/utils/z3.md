# Z3 utilities

Helpers for parsing simple FOL-like strings into Z3 expressions and solving them.

## Files

- Z3_parser.py: Lightweight parser and utilities to build Z3 objects from lists of FOL facts/rules.

## Supported input

- Quantifiers: `ForAll(x, ...)`, `Exists(x, ...)`
- Boolean operators: `AND`, `OR`, `NOT`, `->`, `<->`
- Predicates: `P(x)` / `R(a, b)`
- Comparisons: `=`, `!=`, `>=`, `<=`, `>`, `<`
- Membership: `x IN y` (parsed as predicate `In(x, y)`)
- Literals: integers, floats, and single-quoted strings

## Main entry point

- `prepare_z3_objects(facts, rules, sort_name="U")` returns:
	- `symbols`: shared symbol table
	- `fact_exprs`: list of Z3 Bool expressions for facts
	- `rule_exprs`: list of Z3 Bool expressions for rules

## Example

```text
ForAll(x, (student(x) AND enrolled(x, cs101)) -> passed(x))
```

```python
from src.utils.Z3_parser import prepare_z3_objects
from z3 import Solver, sat

facts = ["student(alice)", "enrolled(alice, cs101)"]
rules = ["ForAll(x, (student(x) AND enrolled(x, cs101)) -> passed(x))"]

symbols, fact_exprs, rule_exprs = prepare_z3_objects(facts, rules)

s = Solver()
s.add(*fact_exprs)
s.add(*rule_exprs)

if s.check() == sat:
		model = s.model()
		print(model)
else:
		print("unsat")
```

## Notes

- This parser is intentionally minimal and does not implement full FOL or rich arithmetic.
- Use data/process.ipynb to normalize Unicode operators and membership ("∈" to "IN") before parsing.
