# Z3 Utilities

Helpers for parsing FOL-like strings into Z3 expressions and verifying logical entailment.

## Files

- **`verifier.py`**: Lightweight parser, symbol manager, and verifier utilities. Parses FOL formulas into Z3 expressions and checks entailment via the SMT solver.

## Supported Input Syntax

| Construct | Syntax |
|-----------|--------|
| Universal quantifier | `ForAll(x, ...)` |
| Existential quantifier | `Exists(x, ...)` |
| Boolean AND | `AND` |
| Boolean OR | `OR` |
| Negation | `NOT` |
| Implication | `->` |
| Biconditional | `<->` |
| Predicates | `P(x)`, `R(a, b)` |
| Comparisons | `=`, `!=`, `>=`, `<=`, `>`, `<` |
| Arithmetic Operators | `+`, `-` *(new)* |
| Membership | `x IN y` → parsed as `In(x, y)` |
| Temporal Constants | `Time830AM` (parsed as `510`), `Time500PM` (parsed as `1020`) *(new)* |
| Duration Constants | `Duration4Hours` (parsed as `240`), `Duration30Minutes` (parsed as `30`) *(new)* |
| Literals | integers, floats, single-quoted strings |

Unicode symbols (`¬`, `∧`, `∨`, `→`, `↔`) are automatically normalized before parsing.

## Main Entry Points

### `parse_formulas(formulas, sort_name="U")`
Parses a list of FOL strings into Z3 Bool expressions.
- **Pre-scans** all input formulas for numeric comparisons to automatically declare numeric symbols as `IntSort()` instead of the default universe sort `U`. This completely eliminates Z3 sort mismatches.

Returns:
- `symbols` — shared `Z3Symbols` symbol table (constants, predicates, functions)
- `formula_exprs` — list of `BoolRef` Z3 expressions

### `verify_with_z3(premises_fol, conclusion_fol, negate_conclusion=True)`
Parses premises and (optionally negated) conclusion, adds them to a Z3 `Solver` with tracking variables, and checks satisfiability.

Returns a dict:
| Key | Type | Description |
|-----|------|-------------|
| `result` | `z3.CheckResult` | `unsat` (entailed), `sat` (counterexample), or `unknown` |
| `proof` | `z3.Expr` \| `None` | Formal Z3 proof object when `unsat` |
| `unsat_core` | `list[str]` | Tracking variable names in the minimal core (e.g. `["p_1", "p_3", "neg_conclusion"]`) |
| `model` | `z3.Model` \| `None` | Counterexample model when `sat` |

### `try_parse_fol(formula)`
Validates a **single** FOL formula without running a full solver check.

Returns `(True, "")` on success or `(False, error_message)` on failure.

### `extract_proof_structure(proof)` *(new)*
Post-order traverses the Z3 formal proof tree recursively, safely checking `z3.is_app(node)` to avoid `Z3Exception`. It extracts all asserted leaf premises and intermediate logical deduction rules (e.g. `mp`, `unit-resolution`), compiling them into a clean, human-readable proof skeleton. Used to guide the LLM CoT generator.

## Example

```python
from src.logic.reasoning.verifier import parse_formulas, verify_with_z3, try_parse_fol

# --- Validate a single formula ---
ok, err = try_parse_fol("ForAll(x, Human(x) -> Mortal(x))")
print(ok)   # True

ok, err = try_parse_fol("ForAll(x Human(x))")  # missing comma
print(ok, err)  # False, "Expected ',', got 'Human'"

# --- Option 1: Parse and solve manually ---
premises = ["Human(Socrates)", "ForAll(x, (Human(x) -> Mortal(x)))"]
symbols, exprs = parse_formulas(premises)

from z3 import Solver, sat
s = Solver()
s.add(*exprs)
if s.check() == sat:
    print("Satisfiable!", s.model())

# --- Option 2: Verify entailment ---
verification = verify_with_z3(
    premises_fol=["Human(Socrates)", "ForAll(x, (Human(x) -> Mortal(x)))"],
    conclusion_fol="Mortal(Socrates)"
)

from z3 import unsat
if verification["result"] == unsat:
    print("Proven! Unsat core:", verification["unsat_core"])
    # ["p_1", "p_2", "neg_conclusion"]
```

## Notes

- This parser is intentionally minimal and does not implement full FOL or rich arithmetic.
- Use `try_parse_fol()` after LLM translation to detect and repair broken formulas before passing to `verify_with_z3`.
- Unbalanced parentheses are auto-corrected during the Unicode-normalization fallback inside `verify_with_z3`.
